      47             : /** This file I/O class supports the Level 5 MAT-File Format of MATLAB.
      48             :  *
      49             :  *  It allows to exchange data between MATLAB and LAMA applications.
      50             :  *
      51             :  *   /code
      52             :  *   save example.mat matrix
      53             :  *   load example.mat
      54             :  *   /endcode
      55             :  *
      56             :  *   /code
      57             :  *   CSRStorge<ValueType> storage;
      58             :  *   storage.readFromFile( "example.mat" );
      59             :  *   storage.writeToFile( "example.mat" );
      60             :  *   /endcode
      61             :  *
      62             :  *  The following topics should be observed:
      63             :  *
      64             :  *  - LAMA can only read and write one single data element from a MATLAB file.
      65             :  *  - The name of the data element is is ignored when reading the element and
      66             :  *    each written element gets the name "LAMA".
      67             :  *  - As the data type is stored for each element in the file, the SCAI_IO_TYPE
      68             :  *    is ignored, i.e. each array/storage is written exactly in the format it is
      69             :  *    and there might be an implicit type conversion during the read.
      70             :  *  - Only the SCAI_IO_TYPE=PATTERN will be handled and in this case no sparse matrix
      71             :  *    values are written, only the row and column indexes
      72             :  *  - The data types LongDouble and ComplexLongDouble are not really supported
      73             :  *    by the Level 5 MAT-File format but are full supported here by using a reserved
